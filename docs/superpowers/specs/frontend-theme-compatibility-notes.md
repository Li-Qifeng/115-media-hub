# 前端日夜主题适配注意事项

本文件记录本地 AI 协作中反复踩到的界面问题，尤其是新增页面、说明块和提示区域的日/夜间模式适配。

## 新增 UI 必须检查日间模式

- 新增页面、弹窗、说明块、提示卡片时，不要只使用深色 Tailwind 类（例如 `bg-slate-900/70`、`text-slate-400`、`border-slate-700`）完成样式。
- 对用户可见的新 UI，优先创建语义类，再在 `static/css/index.css` 里同时声明夜间默认样式和 `html.theme-day ...` 日间覆盖。
- 特别注意说明类内容：标题、正文、边框、半透明背景在日间模式下容易对比度不足，必须明确设置浅色背景、深色标题和可读正文。
- 修改完成后至少检查两个主题下的可读性；如果只是代码验证，也要在最终说明里标注未做浏览器视觉检查。

## 推荐做法

- 说明块使用类似 `settings-info-note` / `settings-info-note-title` 这类语义类。
- 日间模式覆盖应包含：`background`、`border-color`、正文 `color`、标题 `color`。
- 不要把新增说明文字直接长期绑定在深色背景类上，后续维护很容易漏掉日间模式。
